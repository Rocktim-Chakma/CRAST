# Reusable SysCall Feature Extractor

This folder contains the reusable feature-extraction implementation for CRAST.

Unlike the registry-specific wrapper scripts, this extractor can be used for any
registry/class split by passing the input trace directory, output CSV path,
registry name, and class label as arguments.

## File

| File | Purpose |
|---|---|
| `extract_strace_features.py` | Parses raw `strace` output files and converts install-time SysCall behavior into a package-level feature CSV. |

## Typical Usage

```bash
python extract_strace_features.py \
  --input data/raw_traces/npm_benign \
  --output data/features/npm_benign_features.csv \
  --registry npm \
  --label 0
```

For malicious PyPI traces:

```bash
python extract_strace_features.py \
  --input data/raw_traces/pypi_malicious \
  --output data/features/pypi_malicious_features.csv \
  --registry pypi \
  --label 1
```

## Input

The expected input is a folder containing package-level trace folders or
`trace.*` files generated by `strace -ff`.

Example:

```text
sample_package/
├── trace.6297
├── trace.6299
├── trace.6301
└── run.stderr.log
```

## Output

The output is a CSV file where each row represents one package/sample and each
column represents a SysCall-derived feature.

Example columns include:

```text
sample_id
registry
label
total_syscalls
unique_syscalls
failed_syscalls
count_execve
count_openat
count_connect
file_ratio
network_ratio
process_ratio
```

## Relationship to Original-Style Scripts

The original-style and registry-specific scripts make the paper pipeline easier
to audit. This reusable extractor provides the same idea in a cleaner form for
reproduction and reuse.
