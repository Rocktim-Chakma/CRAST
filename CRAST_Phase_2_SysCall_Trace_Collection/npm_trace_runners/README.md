# NPM Trace Runners

This folder contains the shell scripts used for collecting install-time SysCall
traces from NPM packages.

The scripts execute NPM package installation under `strace`, store trace outputs
separately, record stdout/stderr logs, and maintain completed/failed execution
lists. Both original-style and configurable runners are included.

## Files

| File | Purpose |
|---|---|
| `npm_worker_trace_runner.sh` | Original-style worker runner for tracing NPM packages from a worker-specific package list. It preserves the Raspberry Pi worker execution style used during data collection. |
| `npm_worker_trace_runner_configurable.sh` | Configurable version of the worker runner. It supports user-defined package list, package directory, trace directory, log directory, and timeout settings. |
| `npm_archive_trace_runner.sh` | Original-style runner for tracing NPM package archives such as `.tgz` files. It installs archived packages and records install-time SysCall traces. |
| `npm_archive_trace_runner_configurable.sh` | Configurable version of the archive runner. It supports user-defined archive, workspace, trace, and log paths. |

## Runtime Isolation

NPM packages are installed through the Node/npm runtime, not through Python/pip.
Therefore, the NPM tracing scripts use package-specific temporary workspaces and
NPM-specific runtime isolation controls.

The main controls are:

- package-specific workspace directory,
- temporary `HOME` directory,
- package-specific npm cache directory,
- temporary `TMPDIR`,
- fixed execution timeout,
- `strace -ff` child-process tracing,
- separated stdout/stderr logs,
- completed/failed package lists.

Python virtual environments are used for PyPI tracing because PyPI installation
is directly performed through Python/pip. For NPM, the corresponding isolation is
performed through temporary workspace, npm cache, `HOME`, and `TMPDIR`
separation.

## Typical Outputs

A trace run may produce files such as:

```text
trace.<pid>
run.stdout.log
run.stderr.log
completed_packages.txt
failed_packages.txt
```

The `trace.<pid>` files are generated by `strace -ff`, where each file
corresponds to a traced process or child process created during installation.

## Safety Notice

These scripts may execute NPM lifecycle hooks such as `preinstall`, `install`,
and `postinstall`. They should only be run inside a controlled research
environment, preferably on disposable worker machines or virtual machines, with
restricted network access and no personal credentials.

Do not run these scripts on a personal or production machine.
