# PyPI Trace Runners

This folder contains the shell script used for collecting install-time SysCall
traces from PyPI packages.

PyPI packages are installed through Python/pip, so the PyPI runner uses a
package-specific temporary Python virtual environment for installation and
tracing. The installation process is executed under `strace`, and trace files,
stdout/stderr logs, and completed/failed execution lists are stored separately.

## File

| File | Purpose |
|---|---|
| `pypi_venv_trace_runner_configurable.sh` | Configurable PyPI tracing runner. It creates a temporary workspace and Python virtual environment for each package, installs the package through the virtual environment, and records install-time SysCall traces using `strace`. |

## Runtime Isolation

For PyPI tracing, the main package installation command is Python/pip based.
Therefore, a temporary Python virtual environment is used directly for each
package run.

The main controls are:

- package-specific workspace directory,
- temporary Python virtual environment,
- temporary `HOME` directory,
- package-specific pip cache directory,
- temporary `TMPDIR`,
- fixed execution timeout,
- `strace -ff` child-process tracing,
- separated stdout/stderr logs,
- completed/failed package lists.

This differs from NPM tracing because NPM packages are installed through
Node/npm rather than Python/pip. The containment goal is the same, but the
runtime-level implementation is registry-specific.

## Typical Outputs

A trace run may produce files such as:

```text
trace.<pid>
run.stdout.log
run.stderr.log
venv_path.txt
python_version.txt
pip_version.txt
pyvenv.cfg
completed_packages.txt
failed_packages.txt
```

The `trace.<pid>` files are generated by `strace -ff`, where each file
corresponds to a traced process or child process created during installation.

## Safety Notice

PyPI packages may execute setup/build/install hooks during installation. This
script should only be run inside a controlled research environment, preferably on
disposable worker machines or virtual machines, with restricted network access
and no personal credentials.

Do not run this script on a personal or production machine.
